# 浏览器插件安装指南（小白版）

这份文档只讲一件事：**怎么把插件加到浏览器里并用起来**。

## 1. 你手里可能有两种文件

- 情况 A：你拿到的是一个 `zip` 文件（例如 `llm-api-key-checker-extension-v2.x.x.zip`）
- 情况 B：你拿到的是整个项目源码目录

> 两种方式都能安装。推荐新手用「zip 方式」。

## 2. 安装（zip 方式，推荐）

1. 把 `zip` 解压到一个固定目录（不要放在会自动清理的临时目录）。
2. 打开 Chrome 或 Edge。
3. 地址栏输入：`chrome://extensions` 后回车。
4. 打开右上角 **开发者模式**。
5. 点击左上角 **加载已解压的扩展程序**。
6. 选择你刚解压后的文件夹（文件夹内应能看到 `manifest.json`）。
7. 安装成功后，点击浏览器右上角扩展图标即可打开工具。

## 3. 安装（源码方式）

如果你拿到的是源码目录（没有现成 dist）：

1. 安装 Node.js（建议 18+）。
2. 在项目根目录执行：

```bash
npm install
npm run build
```

3. 打开 `chrome://extensions`。
4. 开启 **开发者模式**。
5. 点击 **加载已解压的扩展程序**。
6. 选择项目里的 `dist` 目录。

## 4. 更新到新版本

如果你拿到了新的 zip：

1. 删除旧解压目录（或改名备份）。
2. 解压新 zip 到新目录。
3. 打开 `chrome://extensions`。
4. 在这个插件卡片上点击 **刷新**（循环箭头图标）。
5. 如果插件失效，点一次 **重新加载** 并重新选择新目录。

## 5. 常见问题

### Q1: 看不到“加载已解压的扩展程序”按钮
- 没开开发者模式。先打开右上角 **开发者模式**。

### Q2: 选择目录后报错
- 你选错层级了。必须选到包含 `manifest.json` 的那一层目录。

### Q3: 更新后界面没变化
- 没刷新扩展。去 `chrome://extensions` 点击插件卡片上的刷新按钮。

### Q4: 获取模型失败 / Failed to fetch
- 先确认 Base URL 和 Key 是否正确。
- 再看浏览器控制台错误信息（我们已做端点重试和回退）。

## 6. 隐私说明

- 本插件是本地运行，Key 不会上传到第三方中转服务器。
- 历史记录保存在浏览器本地存储中。
